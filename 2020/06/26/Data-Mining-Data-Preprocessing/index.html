认识数据

数据对象与属性类型

数据集由数据对象组成，一个数据对象代表一个实体；属性是一个数据字段，表示数据对象的一个特征。

标称、二元、序数、数值(区间标度、比率标度)

标称、二元、序数属性都是定性的，仅描述对象特征而不给出实际大小或数目

  1. 标称属性(nominal/categorical attribute)：值为一些符号或事务的名称，代表某种类别、编码或状态，不必具有有意义的序，也不是定量的；在计算机中，可视作枚举(enumeration)的。如头发颜色(黑、棕、淡黄、红色……)，职业(教师、牙医、程序员、农民……)，顾客编号等。
  2. 二元属性(binary/bool attribute)：是一种标称属性，只有两个类别或状态。若两种状态具有同种价值且权重相同，则称该二元属性是对称的，如性别(男、女)；否则如果其状态结果不同等重要，则称不对称的，如艾滋病(阴性、阳性)阳性结果较为重要，一般作为$1$。
  3. 序数属性(ordinal attribute)：可能的取值间具有有意义的序或秩(ranking)，但是相继值之间的差未知；也可以通过把数值属性进行值域划分，离散化为多个有序类别得到。序数属性的中心趋势可以用众数和中位数度量，但不能定义均值。如顾客满意度(满意、中等、不满意)。
  4. 数值属性(numerical attribute)：定量的，即可度量的量，用实数表示，可以是区间标度的或比率标度的
    1. 区间标度(interval-scaled)：用相等的单位尺度度量，除了中心度量众数和中位数度量外还可以定义均值。但是这类属性不具有真正的零点，不能用比率描述不同值的关系，例如温度、日期；
    2. 比率标度(ratio-scaled)：具有固定零点的数值属性，除了中心度量众数和中位数度量外还可以定义均值。可以用比率描述值的关系，例如长度、速度。

离散、连续

  • 离散属性：具有有限或无限可数个值，可以用或不用整数表示，可以是标称的、二元的、序数的、数值的。如果一个属性可能的值集合是无限的，但是可以与自然数一一对应，那么称之为是无限可数的。在实践中用有限位数字表示；
  • 连续属性：不是离散的属性就是连续的，实践中一般用浮点变量表示。

图形显示：分位数图、分位数-分位数图、直方图、散点图

    设属性$X$的$n$个观测按递增排列，定义$f_i$表示存在大约$f_i \times 100\%$的观测值小于$x_i$

    用于观测从一个分布到另一个分布是否存在漂移。假设从一个分布中取得$n$个观测${x_i, i = 1, \cdots, n}$，从另一个分布中取得$n$个观测${y_i, i = 1, \cdots, n}$。两组观测递增排序后，以$(x_i, y_i)$为坐标点简单描点作图，即可得到$q-q$图。

    • 标称数据：可以对于每个标称值，统计其出现的频数/频率进行作图；
    • 数值数据：将值域划分为不相交的连续子域(桶(bucket)、箱(bin))，范围称作宽度(通常等宽)，对落到每个子域内的数据频数/频率进行统计作图。

    将数据点描绘在图上可得散点图，可用于确定两个数值变量间是否存在联系、模式或趋势，例如观察点簇和离群点，或考察相关性。

数据预处理

数据质量包括以下因素

  • 准确性
  • 完整性
  • 一致性
  • 时效性
  • 可信性
  • 可解释性

数据清理

缺失值

许多元组的一些属性可能没有记录值，即存在缺失值，有以下原因可能产生缺失值

  • 被要求提供属性值的人拒绝提供和/或没有所需要的信息；
  • 数据输入者不知道正确的值；
  • 值在稍后提供。

有如下方法处理

  1. 忽略元组：通常在缺少类标号时使用，忽略整个元组，可能丢失剩余有效的属性值，特别是在每个属性缺失值变化很大时效果很差；
  2. 人工填写：费时费力；
  3. 全局常量填充：常量可能被挖掘算法视作一个有趣的概念，并不十分可靠；
  4. 全局中心度量填充：对称数据分布可以使用均值，非对称数据分布应使用中位数；
  5. 同类中心度量填充：填充与该元组类标号一致的数据的中心度量；
  6. 基于推理填充：用回归、贝叶斯、决策树等方法推断缺失值；
  7. 随机填充：在缺失量特别少(通常认为小于1%)的时候，可以随机生成。
  • $3 \sim 6$使数据有偏，填入值可能不正确；
  • 缺失值并不意味着数据有误，例如某项无可填选项时用户自然不填；
  • 实际机器学习工程中，直接删除、众数填充和直接离散化方法用的最多。

每个属性应当有一个或多个关于空值条件的规则，用以说明是否允许有空值，并且/或者说明空值应当如何处理或转换，好的数据库和数据输入设计将有助于在第一现场将缺失值或错误数量降到最低。

噪声

噪声是指被测量的变量的随机误差或方差，可以用数据光滑技术一定程度去除噪声影响

  1. 分箱：通过考察数据的“近邻”来光滑有序数据值，先将数据划分到大小相同的箱中，然后每个箱内进行光滑操作
    • 用箱中心度量光滑：箱内数据计算中心度量(均值、中位数等)，替代箱中数据；
    • 用箱边界值光滑：计算箱中的最大、最小值，箱中每个值替换为最近的边界值。
  2. 回归：用回归方法拟合数据来光滑数据
  3. 离群点分析：用聚类等方法检测离群点，并将其去除

清理过程

数据清理一般通过偏差检测和数据变换两个过程迭代执行

  1. 偏差检测

    • 可以通过元数据发现噪声、离群点和需要考察的不寻常的值。元数据包括
      • 数据类型
      • 定义域
      • 均值、中位数、众数等统计量
      • 数据分布是否对称
      • 数据分布是否符合期望，如数据落在期望的区间
      • 每个属性的标准差
      • 属性间的依赖关系
      • ……
    • 根据唯一性规则、连续性规则和空值规则考察数据
      • 唯一性规则：给定属性的每个值都必须不同于该属性的其他值；
      • 连续性规则：属性的最低和最高值之间没有缺失值，并且所有的值唯一；
      • 空值规则：说明指示空值条件的符号使用，以及如何处理。

  2. 偏差纠正：定义并使用一系列变换来纠正偏差。

数据集成

数据集成是指合并来自多个数据存储的数据，小心集成可以减少数据集的冗余度和不一致。

实体识别

实体识别问题是指如何匹配多个信息源的现实世界的等价实体。当一个数据库的属性与另一个匹配时，需要特别注意数据结构，确保源系统中函数依赖，以及参照约束与目标系统中的匹配。

冗余和相关分析

冗余是指一个属性能由另一个(或另一组)属性导出，属性的命名不一致可能也会导致冗余。冗余可以通过相关分析检测：对于标称数据，可以采用$\chi^2$检验；数值数据可以采用相关系数(correlation coefficient)和协方差(covariance)检测。

标称数据的$\chi^2$检验

假设当前有两个属性$A, B$待检验，$A$包含$c$个不同值$a_1, \cdots, a_c$，$B$包含$r$个不同值$b_1, \cdots, b_r$。以$A$的$c$个属性值为列，$B$的$r$个属性值为行构成相依表，$(A_i, B_j)$表示联合事件$(A = a_i, B = b_j)$如下

$a_1$ $a_1$ $\cdots$ $a_c$ 合计
$b_1$ $o_{11}$ $o_{12}$ $o_{1c}$ $B_1$
$b_2$ $o_{21}$ $o_{22}$ $o_{2c}$ $B_2$
$\cdots$ $\ddots$ $\cdots$
$b_r$ $o_{r1}$ $o_{r2}$ $o_{rc}$ $B_r$
合计 $A_1$ $A_2$ $\cdots$ $A_c$ $n$

那么$(A_i, B_j)$的期望频数为

  • $|A_i|$：$A$取$a_i$的元组个数；
  • $|B_j|$：$B$取$b_j$的元组个数；
  • $n$：总的元组个数。

  • $o_{ij}$：联合事件$(A_i, B_j)$的观测频数(实际计数)；
  • 对$\chi^2$值贡献最大的是观测频数和期望频数很不相同的单元。

$\chi^2$统计检验假设$A, B$独立，具有自由度$(r - 1) \times (c - 1)$，在置信水平$\alpha$下可以查找$\chi^2$分布百分点表得到拒绝假设的值。若计算得$\chi^2$值大于该值，那么可以拒绝$A, B$是独立的假设，也就是说这两个属性是相关的。

数据归约

数据归约(data reduction)旨在得到数据的规约表示，但能够保持数据的完整性。

维规约

维规约(dimensionly reduction)减少所考虑的随机变量或属性的个数。

小波变换：DWT

主成分分析：PCA

属性子集选择

属性子集选择通过删除不相关或冗余的属性减少数据量，找到最小属性集使得数据类的概率分布尽可能地接近使用所有属性得到的所有属性得到的原分布。

通常使用压缩搜索空间的启发式方法(典型的贪心算法)，选择属性的“好”和“差”可以通过统计显著性检验确定，或者是信息增益度量，有以下算法

  • 逐步向前归纳：由空属性集作为归约集，确定原属性集中最好的属性，并在每次的迭代中依次加入剩余属性集中最好的属性；
  • 逐步向后删除：由整个属性集开始，每一步迭代中删除剩余属性集中最差的属性；
  • 逐步向前选择和逐步向后删除的组合：每次迭代中选择一个最好的属性，并删除一个最差的属性；
  • 决策树归纳：由给定的数据构造决策树，不出现在树中的所有属性假定是不相关的，出现在树中的属性形成规约后的属性子集。

回归和对数线性模型：参数化数据规约

回归和对数线性模型用于近似给定的数据，在回归方法中用一个(或一组)属性拟合另一个属性从而降低维数；而对数线性模型近似离散的多维概率分布，对于给定的元组集合，每个元组视作$n$维的点，用对数线性模型基于维组合的一个较小子集，估计多维空间中每个点的概率，使得高维数据可以由较低维空间构造。

数量规约

数量规约(numerousity reduction)用替代的、较小的数据表示形式来替代原数据。

直方图

用桶对属性值进行划分，包括等宽和等频两种方法，得到桶聚合，然后将桶作为数据的压缩表示，从而减少元组数目。

聚类

类似分桶，通过聚类方法将对象划分为群或簇，用数据的簇代表替换实际数据。该方法的有效性依赖于数据的性质。

抽样

用数据的小得多的随机样本子集代表大型数据集。抽样方法的优点是，算法复杂度仅与样本集大小有关，而与数据集大小无关。通常方法有

  • $s$个样本的无放回简单随机抽样；
  • $s$个样本的有放回简单随机抽样；
  • 簇抽样：将元组放入不同的簇中，每个簇中进行抽样，例如基于邻近度定义簇；
  • 分层抽样：数据集被划分为互不相交的部分，即“层”，对每一层进行简单随机抽样。特别是当数据倾斜时，可以帮助确保样本的代表性。

数据立方体聚集

数据压缩

数据压缩(data compression)使用变换得到原数据的规约或“压缩”表示，维规约和数量规约可以视作某种形式的数据压缩。

数据变换

用于将数据变换或统一成适合于挖掘的形式，包括

  • 光滑：去掉噪声，包括分箱、回归和聚类；
  • 属性构造：由给定属性构造新的属性，并添加到属性集中；
  • 聚集：对数据进行汇总或聚集；
  • 规范化：把单属性数据按比例缩放，落到特定的小区间；
  • 离散化：将数值属性的原始值用区间标签或概念标签替换，这些标签可以递归地组织成更高层概念，导致数值属性地概念分层；
  • 由标称属性产生概念分层：将具有一定概念的属性泛化到更高的概念层。

作用是

  • 避免异常点：比如对连续变量进行份桶离散化
  • 可解释性或者需要连续输出：比如评分卡模型中的iv+woe
  • 使得原始数据的信息量更大：比如log/sqrt变换

规范化

试图赋予所有属性相等的权重，用于减少对度量单位的依赖性。

min-max(最小-最大)规范化

对原始数据仅线性变换，仅跟最大、最小值的差别有关，只是一个去量纲的过程，将落在$[min, max]$的属性值$a$映射到$[MIN, MAX]$中

z-score(零均值)规范化

缩放和所有点都相关，数据相对分布不会改变，集中的数据标准化后依旧集中。

  • 基于属性$A$的均值$\overline{A}$和方差$\sigma_A$

  • 或者是$A$的均值绝对偏差(mean absolute deviation)$s_A$代替$\sigma_A$

$\sigma_A$与$s_A$分别代表$A - \overline{A}$的$L_2, L1$范数。

min-max与z-score的关系与对比

  • 异常点角度：特征数据上下限明显异常的时候使用z-score规范化方法，简单min-max规范化会造成数据差异模糊，整体方差下降；
  • 分布角度：使用z-score规范化之前，要求数据需要近似满足高斯分布，不然会改变数据的分布，尤其是对数据分布有强假设的情况下；
  • 上线变动角度：min-max规范化在上线的时候需要考虑上下约束界是否需要变动，z-score规范化则不需要考虑变动；
  • 值域范围角度：min-max规范化对数据范围约定较为固定，而z-score规范化的输出上下界则不定；
  • 模型角度：一般涉及距离计算，协方差计算，数据满足高斯分布的情况下用z-score规范化，其他min-max规范化或其他变换。

数据离散化

分箱

基于指定箱个数的自顶向下的分裂技术，例如用等宽或等频分箱后，用均值或中位数替换箱中的每个值，从而离散化。对用户指定的箱的个数很敏感，且容易受离群点影响。适用于小范围连续数据内不存在逻辑关系的数据，比如31岁和32岁之间不存在明显的差异，可以归为一个箱中。

聚类

考虑属性$A$的分布及其数据点的邻近性，将属性值划分为多个簇，可以产生高质量的离散化结果。

决策树

主要思想是，通过选择划分点使得一个给定的结果分区包含尽可能多的同类元组，将属性离散化。这种离散化可以形成$A$的概念分层。

相关分析

例如，基于$\chi^2$的离散化方法$ChiMerge$采用自底向上的策略，递归找出最邻近的区间进行合并。基本思想是，对于精确的离散化，相对类频率在一个区间内应当完全一致。因此，若两个邻近区间具有类似的类分布，那么可以进行合并。

概念分层

用于将数据变换到多个粒度层，适用于有一定规律的类别数据，邮政编码，手机号等，产生方法有

  • 由用户或专家在模式级显式说明属性的部分序；
  • 通过显式数据分组说明分层结构的一部分；
  • 说明属性集但不说明它们的偏序：可以根据给定属性集中，每个属性不同值的个数，自动产生概念分层。属性不同值个数越少，其所在概念分层结构中所处的层次越高。
  • 只说明部分属性集：依照属性的语义相关性进行分层。
